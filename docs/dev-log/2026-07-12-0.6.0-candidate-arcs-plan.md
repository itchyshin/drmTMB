## Arc 1 — REML: bring Gaussian REML up to ML parity

**Shinichi:** "REML is only for Gaussian, but it's not as capable as ML — that's to be done."

- **State:** Gaussian/biv-Gaussian REML exists (`REML = TRUE`) but scope-restricted — mean-side phylo at q1; sigma-side/q2/q4 partial; the structured providers `spatial()`/`animal()`/`relmat()` are rejected under REML (design 211/221). Non-Gaussian REML is blanket-rejected + under an active forbidden-wording guard (SR159), and is genuinely hard (finite-sample SD-skew blockers, `2026-07-06-next-arc-ultraplan`).
- **The arc = close the Gaussian REML↔ML capability gap**, NOT non-Gaussian REML.
- **Splits:** (1a) Gaussian REML for the structured providers (spatial/animal/relmat) to ML parity; (1b) q2/q4 coupled mean+scale REML completion; (1c) *[separate, likely POST-0.6.0]* non-Gaussian REML — needs a design decision to lift SR159 + the hard finite-sample work.
- **Why it matters:** REML's payoff is *unbiased variance components* — it pairs with Arc 2 (RE) for small-cluster-n honesty.
- **Priority:** MEDIUM. Do 1a (+1b) for 0.6.0; hold 1c.

## Arc 2 — Random effects: at least an intercept everywhere, one slope each

**Shinichi:** "some don't take random effects… want at least random intercept accepted, and each random effect one random slope."

- **State:** 5 families take NO random effects (binomial, cumulative_logit, skew_normal, tweedie, zero_one_beta); `sigma`-RE rejected except gaussian (full) + nbinom2 (int-only); shape/inflation dpars (nu/zi/hu/zoi/coi) are fixed-effect-only almost everywhere; random slopes are partial.
- **Splits:** (2a) random INTERCEPT on `mu` for the 5 no-RE families — the concrete, high-impact win; (2b) one random SLOPE per existing RE where missing; (2c) `sigma`-RE broadening (harder — variance-of-variance identifiability).
- **Why it matters:** random intercepts are the single most-used feature for applied eco/evo. "At least an intercept everywhere" is a bounded, systematic, headline 0.6.0 deliverable.
- **Priority:** HIGH — recommended first. It's the foundation Arcs 1 and 3 build on.

## Arc 3 — Structured dependency across distributions

**Shinichi:** "structural dependency is not built for all the distributions."

- **State:** 8 families have NO structured provider; most others expose one provider on one dpar. Full richness (multiple providers × μ±σ) only for gaussian, nbinom2, poisson, biv-Gaussian location.
- **Splits:** by provider — extend `phylo()`/`relmat()` (the headline eco/evo pair) to more families first; `spatial()`/`animal()` next. Or by family tier. **Depends on Arc 2** (structured effects are structured RE).
- **Priority:** MEDIUM-HIGH. Phylogenetic/spatial is a signature eco/evo feature, but *broad* coverage is a large surface — for 0.6.0, target a defined subset (e.g. phylo+relmat on μ for the common count/continuous families), state the rest as planned.

## Arc 4 — Intervals: test what's there, then coverage

**Shinichi:** "some of the intervals are not fully tested — that's why we want to do that."

- **State:** Wald/profile/bootstrap exist; profile is the featured method (D-12). Coverage is largely untested — 0 `supported` cells, `interval_status` blocked/planned for most structured-RE rows. This is the same axis as the deferred DG4/DG5.
- **Splits:** (4a) validate the existing intervals per family (Wald↔profile agreement, finite/correct at a known DGP point — the DG4 "interval-feasible" bar); (4b) the calibrated-COVERAGE campaign (Totoro/DRAC, the DG5 bar) — reuse the DG3 harness pattern from this arc.
- **Why it matters:** trustworthy intervals are core to "usable." High honesty leverage.
- **Priority:** HIGH — recommended second. 4a is bounded; 4b is a compute campaign.

## Arc 5 — Missing predictors (`mi()`)

**Shinichi:** "the last one."

- **State:** mi() covers 5 families (gaussian broad impute catalogue; poisson/binomial/nbinom2/beta one binary each); extensive prior MD7/MD8 slice work (brain). 13 families reject mi() predictors; multiple/non-binary predictors are planned.
- **Splits:** (5a) more families (one binary predictor); (5b) multiple missing predictors; (5c) non-binary / structured missing predictors.
- **Priority:** LOWER (Shinichi: "last one"). Likely a small extension for 0.6.0 or post-0.6.0.

## Recommended sequence toward 0.6.0

Not all five for 0.6.0. Dependency + leverage argue for:

1. **Arc 2a (RE intercepts everywhere)** — highest usability leverage; the foundation for 1 and 3.
2. **Arc 4a (validate existing intervals)** — honesty; bounded.
3. **Arc 1a (Gaussian REML structured parity)** — pairs with RE for small-n variance components.
4. **Arc 3 (structured phylo/relmat on a defined subset)** — signature feature, scoped.
5. Coverage campaign (**Arc 4b**) + **Arc 5** + non-Gaussian REML (**1c**) — POST-0.6.0 or opportunistic.

Each arc should run the same discipline this one used: symbolic/interface first → freeze → per-family batches with DG2/DG3-style evidence → honest-scope docs → fresh adversarial CP-review defaulting NOT-DONE. Heavy campaigns → Totoro/DRAC. `/ask-brain` before deriving; write durable decisions to files.
