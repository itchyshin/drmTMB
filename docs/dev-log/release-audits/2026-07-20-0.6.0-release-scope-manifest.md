# drmTMB 0.6.0 — release-scope manifest

**Date:** 2026-07-20 · **Basis:** `origin/main` @ `ac378024` (post-D2) · **Author:** Claude lane
(release evidence & truth-freeze arc).

This is the durable, single-place statement of what drmTMB 0.6.0 promises and does not. It is built
from the capability ledger (`docs/dev-log/dashboard/capability-ledger/cells.tsv`, verified consistent:
`capability_ledger.py --check` OK across 30 generated outputs, unittest 37/37, runtime 18 routes
G0=G1=G2=0). It reconciles the ledger with README, `_pkgdown.yml`, ROADMAP, NEWS, and
`capability-and-limits.Rmd` (all aligned in the same PR as this manifest).

**Scope note.** This freeze reconciles *claims*; it does **not** bump `DESCRIPTION` Version to a
release number (Phase 20 / issue #61), add capabilities, or run compute.

**Every inference claim below carries seven fields:** estimator (ML/REML) · interval method · tested
family + model structure · deployment floor / tested N,M domain · evidence tier · known caveats ·
explicit negative space.

## 2. Implemented but NOT inference-certified

- `interval_feasible` (44) and `diagnostic_only` (70): fit + extractor work; interval coverage not
  established. `point_fit_recovery` (176): point recovery at a known DGP, no interval claim.

### 2a. Zero-one-beta (mc-0575) — fenced, generator-qualified, NO compute

**Claim:** `inference_ready_with_caveats`, ML, ML-Laplace profile interval for the natural-scale slope
SD, floor `M >= 16`, **under the generator as executed only.** Coverage 0.9292 / 0.9400 / 0.9517 at
`M = 16 / 32 / 64` (all-attempts, /1200).

**The defect.** The DGP allocated a fixed 15% structural boundary mass with interior draws from a beta
density, but the interior draws leaked machine-exact ones — **50, 87, 193** across the M=16/32/64 banks
— because the precision parameter drives the beta shape toward a boundary in the upper tail. So the
fitted data carried slightly more than 15% observed boundary mass.

**Why the leak is not ignorable — verified from the retained artifact** (`arc4c-raw.tsv`, this arc,
2026-07-20):

| M | full-sample cov (/1200) | clean-subset cov (n) | leak-subset cov (n) | clean interval-centre | leak interval-centre |
|---|---|---|---|---|---|
| 8 | 0.9133 | 0.9118 (1179) | 1.0000 (21) | 0.531 | 0.694 |
| 16 | 0.9292 | 0.9270 (1151) | 0.9796 (49) | 0.516 | 0.559 |
| 32 | 0.9400 | 0.9400 (1117) | 0.9398 (83) | 0.504 | 0.526 |
| 64 | 0.9517 | 0.9502 (1024) | 0.9602 (176) | 0.498 | 0.516 |

The leak-containing replicates carry **systematically larger** slope-SD estimates at every M
(interval-centre proxy; `sd_hat` itself is NA — see §6). So `invalid_interior` is correlated with the
estimand, and dropping leak replicates conditions on an outcome-related variable. The near-identity of
full-sample and clean-subset coverage is an arithmetic consequence of the small leak fraction
(`p_all − p_clean ≡ leak_frac × (p_leak − p_clean)`; at M=64, 0.1467×0.0100 = 0.0015 = observed), **not**
evidence that the leak did not bias coverage. **The effect under the intended (strictly-interior)
generator is unquantified and cannot be recovered from the retained campaign.**

**Disposition.** One of three independent promotion reviewers withheld this cell on exactly this
ground; it promotes under the frozen two-withhold rule and the caveat travels with it (ledger,
`capability-and-limits.Rmd`, NEWS). **Named next gate:** a deterministic strictly-interior-generator
rerun before any exactly-15%-boundary claim — a separate future arc requiring compute approval; not
part of 0.6.0.

## 5. Known limitations accepted for 0.6.0 (documented, not deferred)

- **#710 numerical-stability edges (6, `bug`-labelled).** Read in full: all are edge-case hardening —
  opaque NaN instead of a clean "not PSD" error on pathological large-scale/degenerate meta `V` (#1,#4);
  start-value quality only, optimizer recovers (#2,#3); tail-underflow NaN abort in missing-covariate
  imputation (#6, CONFIRMED). 5 of 6 are PLAUSIBLE (static review, unbuilt). **None is a silent wrong
  answer on a valid in-domain input.** **Finding #5** (profile-endpoint inner solves inherit the
  caller's `iter.max`, so a user-set *tight* budget could silently bias a profile CI inward) touches
  the featured profile-CI capability and is flagged for a Fisher look — but it fires only outside the
  default/tested domain.
- **Regression-`rho12` interval** — `corpairs(conf.int=TRUE)` returns no bounds for `rho12 ~ x`
  (`conf.status = "newdata_required"`); only `rho12 ~ 1` yields a profile interval. Found in D2 smoke.
- **#740** dense `chol2inv` pedigree A⁻¹ — works, doesn't scale.
- No cell is coverage-verified at the strictest bar.
