# After Task: v0.4.0 capability-honest release prep (Arcs 0–4)

## 1. Goal

Take drmTMB from the 0.3.0 release to a **release-ready 0.4.0** (staging toward
v1.0) on the principle of **capability honesty**: replace the misleading
"104/104 = 100%" headline with an honest, evidence-tiered picture of the whole
capability surface, align the ledgers, harden the surface where cheap, and reach
a clean `R CMD check --as-cran`. Branch: `drmtmb/v0.4.0-release-prep`.

## 2. Implemented

- **Arc 0 — full-scope capability census + mission-control map.** Enumerated all
  **668** capability cells (18 families × distributional parameter × effect type ×
  dimension × q-gate × estimator), evidence-tiered, via an ultracode Workflow
  (18 per-family enumerators + 18 adversarial verifiers). Consolidated to
  `docs/dev-log/dashboard/capability-census/_master.tsv` and a self-contained
  `capability-map.html`. Two verifier-flagged errors corrected (a bivariate-REML
  scale-only mis-classification; the nbinom2 mis-wire below).
- **Arc 1 — four-ledger alignment.** README / NEWS / ROADMAP / known-limitations /
  `_pkgdown.yml` / DESCRIPTION aligned to 0.4.0 with the **corrected REML boundary**
  (an earlier draft had it backwards) and the 5→8 `inference_ready` anchor fix, all
  verified against `R/drmTMB.R`.
- **Arc 2 — capability hardening.**
  - Fixed a **silent wrong-answer bug** the census surfaced: `nbinom2`
    `sigma ~ phylo/spatial/animal/relmat` applied the structured effect to the
    *mean* predictor (`model_type == 7` lacked the `phylo_mu_dpar == 1` branch that
    beta `model_type == 10` has). Mirrored beta's dispatch; added a recovery test +
    mis-wire regression guard.
  - Ran a **coverage campaign on Totoro** (384-core lab server): unstructured
    non-Gaussian fixed-effect **mean** intervals (binomial/poisson/beta/nbinom2,
    incl. rare-event and low-count stress) and **nbinom2 location-scale** intervals
    are calibrated (Wald, finite-rate ≈ 1.0, near-nominal); beta location-scale
    calibrated for interior proportions. Promoted the 6 validated cells in the
    census.
- **Arc 3 — pkgdown + limitations page.** New user-facing vignette
  `capability-and-limits.Rmd` (three honest tiers, at-a-glance table, a named
  alternative for every rejected route). pkgdown polish (navbar wiring, dropped a
  duplicate reference entry). `check_pkgdown()` clean.
- **Arc 4 — CRAN readiness.** `R CMD check --as-cran` = **0 errors / 0 warnings /
  0 notes** (cleaner than 0.3.0's 0E/0W/1N).

## 9. What Did Not Go Smoothly

- The nbinom2 **scale map sign** was guessed wrong (`exp(+2)` vs `exp(-2)`); the
  smoke test caught it. Lesson reinforced: **write the symbolic-alignment table
  first** (it is now in the artifact README).
- Totoro setup failed twice: rsync carried the Mac-built `init.o` (wrong format for
  Linux → link error), and a stray `.libPaths()` in a shell string. Both fixed.
- The first cluster connectivity sweep failed — the **Bash sandbox** blocked outbound
  SSH; `dangerouslyDisableSandbox` over the live ControlMaster sockets works.

## 10. Known Residuals

- **Release trigger is user-gated:** push branch / open PR / merge / tag `v0.4.0`
  await explicit approval (6 commits sit on `drmtmb/v0.4.0-release-prep`, unpushed).
- **Deferred thin-hole hardening** (recovery-only cells, stay recovery-only): add
  real recovery assertions to `zi_poisson zi~spatial`, `beta ~ animal` (mu/sigma),

  remains a documented mild floor, not resolved.
