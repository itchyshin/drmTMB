# After Task: 35-Map Slice 15 Phylogenetic q4 TMB Probe

## Goal

Add a hidden TMB parameterization scaffold for the constant bivariate
phylogenetic q=4 prior before exposing fitted `sigma1` or `sigma2`
phylogenetic terms.

## Implemented

The template now has a hidden `model_type == 93` branch that reconstructs
endpoint-major q=4 phylogenetic latent effects, builds
`Sigma_phylo = D R D` from four `log_sd_phylo` values and six
unstructured-correlation parameters, and evaluates the matrix-normal prior
against the sparse augmented tree precision. `theta_phylo` is present as a
mapped-off default parameter for ordinary fits.

## Mathematical Contract

The hidden branch implements:

```text
U ~ MatrixNormal(0, Q_aug^{-1}, Sigma_phylo)
Sigma_phylo = D R D
```

where `U` is stored endpoint-major and `R` is generated by TMB's q>2
`UNSTRUCTURED_CORR_t` parameterization.

## Tests Of The Tests

The new q=4 phylogenetic TMB probe compares the objective against
`drm_phylo_correlated_precision_nll()`, checks finite gradients, and verifies
reported SDs, `theta_phylo`, q=4 correlation matrix, q=4 covariance matrix, and
quadratic matrix. Neighboring ordinary q4 probe tests initially failed because
manual specs lacked the new mapped-off parameter; Curie fixed those helpers and
the broader focused set passed.

## Consistency Audit

The design note explicitly says this is a hidden probe only. Public docs still
describe phylogenetic q=4 fitting as planned; this slice does not add
`sigma1`/`sigma2` `phylo()` formula support, six phylogenetic `corpairs()` rows,
or a runnable q4 article example.

## What Did Not Go Smoothly

Adding a new TMB parameter surfaced manual tests that bypass the normal
`drmTMB()` spec finalization path. That was a useful failure because it forces
hidden probes to share the same mapped-parameter defaults as ordinary fits.
