# After Task: Binomial GLM Parity Comparator Artifact

## Goal

Bank the first executable Phase 19 comparator artifact without promoting
release readiness, speed, interval calibration, random effects, structured
effects, bivariate or mixed responses, or the Julia bridge.

## Implemented

Generated a small deterministic fixed-effect Bernoulli/binomial comparator
artifact under
`docs/dev-log/comparator-results/2026-06-16-binomial-glm-parity/`.
The artifact uses the Phase 18 `binomial_fixed_effect` writer with both
supported response encodings, 0/1 rows and `cbind(successes, failures)` rows,
`n = 320`, `n_rep = 3`, seed `20260616`, and one core.

The committed artifact includes CSV tables for aggregate summaries,
replicate-level summaries, the run manifest, failures, Wald intervals,
diagnostic Wald coverage, and `stats::glm()` parity. It also includes a README
with the source SHA, dirty-state note, R/package versions, platform, seed, row
counts, maximum `drmTMB` versus `stats::glm()` differences, and interpretation
label `parity`.

Updated the Phase 19 comparator design note, release-readiness checklist, and
mission-control dashboard so they point to this artifact while still marking
the broader comparator programme and release gate as unfinished. Updated
`tools/start-mission-control.sh` so comparator-result directories are copied
into `/tmp/drm-dashboard` alongside after-task notes and design evidence.

## Mathematical Contract

No likelihood, parameterization, or formula grammar changed. The checked
contract remains:

```text
Y_i ~ Binomial(n_i, mu_i)
logit(mu_i) = eta_i = X_mu[i, ] beta_mu
```

The artifact supports only this claim: `drmTMB` fixed-effect
`stats::binomial(link = "logit")` fits match `stats::glm()` on overlapping
fixed-effect logit likelihoods for the two supported response encodings.

## Tests Of The Tests

This is an evidence-artifact slice, not a new package-code slice. The
artifact writer had already been tested in the Phase 18 binomial evidence PR.
Here the test of the artifact is the independent `stats::glm()` comparator
table plus the manifest/failure ledger. All six target fits returned `ok`, all
six reported `pdHess = TRUE`, and the failure table is header-only.

## What Did Not Go Smoothly

The first dashboard JSON patch matched broad `"planned"` fields and touched an
unrelated visual phase plus the HSquared lesson row. The diff review caught it
before validation; those accidental edits were reverted, and the validator then
confirmed the intended counts.

One metadata command tried to read the package version with
`utils::packageDescription(".")`, which is not the right way to read a source
tree DESCRIPTION file. The committed README uses the corrected DESCRIPTION
read plus loaded package versions.

## Team Learning

Grace's evidence-link rule needs the served dashboard, not just static JSON.
When dashboard rows point to artifact directories, `tools/start-mission-control.sh`
must mirror those directories so browser checks and issue links can reach the
same evidence.

## Known Limitations

`n_rep = 3` is intentionally tiny and supports parity smoke language only. The
Wald coverage table is diagnostic; it does not support interval-calibration
claims. The artifact does not cover binomial random effects, structured
effects, bivariate responses, mixed responses, proportions with weights, speed,
or the Julia bridge.

No R package code changed in this slice, so `devtools::test()` and
`devtools::check()` were not rerun here. `pkgdown::check_pkgdown()` remains
outside this slice and has a known Claude-owned penalty/MAP navigation blocker.
