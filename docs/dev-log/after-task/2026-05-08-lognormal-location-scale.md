# After Task: Lognormal Location-Scale Family

## Goal

Add the first positive continuous response family while preserving the
package's one-formula-per-parameter contract and keeping lognormal support
fixed-effect, univariate, and clearly documented.

## Mathematical Contract

For observation `i`, the implemented model is:

```text
log(y_i) | mu_i, sigma_i ~ Normal(mu_i, sigma_i^2)
mu_i = X_mu[i, ] beta_mu
log(sigma_i) = X_sigma[i, ] beta_sigma
sigma_i = exp(X_sigma[i, ] beta_sigma)
```

The TMB likelihood is evaluated on the original positive response scale:

```text
log f(y_i) = log Normal(log(y_i) | mu_i, sigma_i^2) - log(y_i)
```

The response-scale arithmetic mean is:

```text
E[y_i] = exp(mu_i + sigma_i^2 / 2)
```

Matching public syntax:

```r
drmTMB(
  bf(biomass ~ habitat, sigma ~ treatment),
  family = lognormal(),
  data = dat
)
```

`predict(fit, dpar = "mu")` returns log-scale location. `sigma(fit)` returns
the standard deviation of `log(y)`. `fitted(fit)` returns the arithmetic
response mean.

## Tests Of The Tests

- The independent likelihood test compares the fitted objective to
  `stats::dlnorm()` at the fitted coefficients, including the lognormal
  Jacobian.
- Parameter-recovery tests check the link-scale `mu` and `sigma` coefficients.
- Factor-predictor tests check model-matrix construction for `mu` and `sigma`.
- Small- and large-`sigma` tests check scale edge cases without relying on a
  long stochastic simulation.
- Missing-row tests confirm complete-case filtering happens before positivity
  checks, so a zero response on a dropped row does not poison the fit.
- Negative tests cover unsupported `nu`, `sd(group)`, `meta_known_V(V = V)`,
  `mvbind()`, duplicate `sigma`, missing response, and no complete
  observations.

note.

## Known Limitations

- `lognormal()` currently supports fixed effects only.
- Random effects, random-effect scale formulae, known sampling covariance,
  phylogenetic terms, spatial terms, bivariate lognormal models, and mixed
  composed families involving lognormal responses are not implemented.
- `mu` is log-scale location for this family, so users should use `fitted()` if
  they want the arithmetic mean on the original response scale.
