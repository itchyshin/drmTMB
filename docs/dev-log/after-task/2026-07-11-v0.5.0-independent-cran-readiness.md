## 3a. Decisions and Rejected Alternatives

- Claude's broader HOLD argument was not accepted as a technical release blocker. Permanence, publicity, and the continuing maintenance commitment are real maintainer considerations, but they do not make an honestly documented experimental package mechanically unready for CRAN.
- The earlier valgrind and rchk failures were not rerun. The Codex handover records that valgrind never reached `drmTMB` because an `emmeans` Suggests build failed, while rchk findings were confined to TMB framework headers. Neither is evidence of a package-owned defect.
- The local spelling-output NOTE was not treated as a CRAN blocker. With `NOT_CRAN=true`, `spelling::spell_check_test()` deliberately writes a comparison file and prints the report-only word list; CRAN leaves `NOT_CRAN` unset, so the test removes the comparison file and exits with `All Done!`.
- The DOI URL-checker 403 was not treated as a broken citation. Crossref resolves `10.1198/0003130032369` to "Raindrop Plots"; the publisher endpoint blocks the automated request.
- After all three sanitizer jobs passed, the annotated `v0.5.0` tag was force-updated from stale commit `09d44c7c` to audited commit `095409c0`, as required by the release instructions. The existing GitHub release remained attached and the retag triggered the full three-OS release matrix.
