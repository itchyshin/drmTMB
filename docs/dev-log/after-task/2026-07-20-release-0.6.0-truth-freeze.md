## 3a. Decisions and Rejected Alternatives

- **Distinguish dev version from latest release, not "replace 0.5.0 everywhere."** The recon's
  "stale 0.5.0" flag was a simplification: some `0.5.0` references (first-release fact, `v0.5.0`
  install tag, versioning-philosophy discussion) are historically correct. Only the *current-version*
  claims were stale. Rejected a blind find-replace.
- **Per-family floors, not one.** Writing the manifest surfaced that the five promoted mu-slope cells
  have **three distinct designs** — skew-normal/Tweedie/zero-one-beta at M>=16 SD 0.50 (ML-Laplace);
  binomial at M>=32 SD 0.6; cumulative-logit at M>=40 SD 0.50 (AGHQ+Cox-Reid). My first draft (and the
  first vignette row) generalized "M>=16 per family" — an overclaim. Corrected in both surfaces.
- **Zero-one-beta: keep tier, fence hard, no compute** (plan §10). The strictly-interior-subset
  comparison is not evidence the leak was harmless — the leak correlates with the estimand (leak
  replicates carry larger slope-SD estimates, verified from `arc4c-raw.tsv` this arc), so the
  near-identical clean-vs-full coverage is an arithmetic identity, not a null result. Claim stated only
  "under the generator as executed"; the rerun is a named next gate requiring compute approval.
- **Correcting comments over body rewrites** for #59 and #61 — preserves issue history.
- **Did not bump DESCRIPTION Version** — freeze scope, not a release.

## 9a. Zero-one-beta rerun — adjudicated against (no compute)

Shinichi opened the door to using Totoro for the strictly-interior rerun. I confirmed Totoro reachable
(384 cores, live socket), wrote a prospective compute spec (freeze everything except the interior
sampler; redraw interior draws strictly to (0,1); retain `sd_hat`), and put it through fresh
Fisher/Noether/Rose plan review **before any compute**. Outcomes:

- **Fisher (the decisive one):** there is NO strictly-interior sampler that faithfully renders the
  intended beta. The leaked draws sit at `shape2 ≈ 0.037`, where the intended beta places ~16–69% of
  its mass within one machine ULP of the boundary (`ε^shape2`, confirmed analytically this arc). The
  intended law genuinely assigns real probability to a region double precision cannot represent as
  interior. So a rerun answers a convention-dependent question, not "coverage under the intended
  design." Fisher's probe: `scratchpad/leak_shape2_probe.R`.
- **Noether (spec consistency):** frozen-value table matched source; flagged that my §4 gate had dropped
  one S0 rule and the M=8 scope was ambiguous — both would have been fixed before running.
- **Rose (scope/provenance):** raised a **NOT READY / "the campaign never ran"** alarm (cells.tsv =
  `point_fit_recovery`, artifact missing). **Verified against ground truth: FALSE ALARM** — mc-0575 IS
  `inference_ready_with_caveats`, the artifact exists, and commit `13667138` (#800) certified it; Rose
  read a stale pre-Arc-4c version. Rose's other findings (four thread pins not one, redraw-only-leaked,
  log shape2/eta) were valid and would have improved the spec.

**Decision (Shinichi): don't run.** Because no faithful strictly-interior rendering exists, the
generator-qualified fence is the *principled terminal characterization*, not a deferred debt. This is a
better outcome than a rerun would have produced — it converts a caveat into a proved boundary. Recorded
in manifest §2a/§6 and the reader-facing vignette caveat. Zero compute.
