# drmTMB 0.5.0 — Shipped-Docs Accuracy Audit (pre-CRAN)

_Generated 2026-07-11 by a 99-agent Workflow: one auditor per shipped surface vs the code-verified capability surface, each finding adversarially verified against the live gates, then consolidated. 37 surfaces, 61 raw findings, **60 confirmed** (2 blocker, 8 high, 10 medium, 40 low)._

## Executive summary

The shipped documentation surface is broadly honest and, consistent with the deliberate bounded 0.5.0 framing, errs overwhelmingly toward UNDER-claiming: of ~40 consolidated discrepancies only two are genuine tier overclaims (both low), and no doc leads a user to over-trust an interval on an unsupported model — the CRAN-dangerous direction. The material problems are concentrated in one surface, vignettes/cross-family.Rmd, which carries the single blocker plus a cluster of high/medium errors: the Julia cross-family accessor contract is inverted (the vignette denies the real exported rho_latent() and prescribes rho12()/confint(parm="rho12") calls that abort), and its worked bf() example passes rho12 = ~ 1, which errors. A systematic stale-version problem (v0.1.3, v0.3.0, "0.4.0 development", future-tense 0.5.0) is spread across the README, _pkgdown navbar, NEWS.md, and the getting-started vignette — all reviewer/user-visible at first CRAN submission. The largest but lowest-severity pattern is the consistent labeling of implemented recovery/diagnostic-grade non-Gaussian structured routes as "planned"/"not available" across ~15 surfaces, plus the deprecated sd_phylo*() spelling taught as current. Net: the surface is safe on capability honesty but ships broken cross-family examples and wrong version stamps that must be fixed before submission.

## Cross-cutting themes

- Cross-family (Julia xfam) accessor contract is systematically wrong in vignettes/cross-family.Rmd — the worst-hit surface: it inverts rho_latent()/rho12(), prescribes rho12() and confint(parm="rho12") calls that error, ships a bf() with rho12 = ~ 1 that aborts, narrates a coef() structure the object lacks, and separately understates that the R route actually ships in 0.5.0. One blocker + ~4 high/medium errors in a single vignette.
- Stale version stamps on every release-facing surface: v0.1.3 (drmTMB.Rmd:35), v0.3.0 (README install pin), '0.4.0 development' (README banner L53, _pkgdown navbar L7 site-wide, NEWS L94-95 'version moves to 0.4.0'), and future-tense 'will be numbered 0.5.0' (README L58). Six stamps, all should read 0.5.0/experimental — the highest CRAN-visibility risk.
- Deprecated sd_phylo*() spelling taught as current across 5 surfaces (which-scale.Rmd, drmTMB.Rmd:122, phylogenetic-models.Rmd:344, large-data.Rmd:87), with which-scale.Rmd L786-790 reversing the deprecation direction entirely. Same stale-form class as tau / meta_known_V() / gr().
- Systematic UNDER-claiming of recovery/diagnostic-grade non-Gaussian structured routes: ~15 surfaces (capability-and-limits, implementation-map, distribution-families, formula-grammar, robust-student, animal-models, spatial-models, relmat-known-matrices, count-nbinom2, proportion-beta-binomial, plus spatial()/nbinom2() roxygen) list implemented single-provider routes — gamma relmat-mu, beta animal, student spatial-mu/phylo-nu, poisson/nbinom2 mu+sigma, zi~spatial, truncated_nbinom2 relmat-hu, cumulative_logit phylo-mu — as 'planned' or 'not available'. Safe direction, but affirmatively false and internally inconsistent with the SSOT and each doc's own Poisson/NB2 treatment.
- The nbinom2 structured-sigma routing fix (0.4.0, src/drmTMB.cpp:3303) was not propagated to docs: NEWS.md L89-93 even contradicts its own Bug-fixes section 40 lines above; implementation-map, distribution-families, and R/family.R nbinom2() roxygen all still call structured sigma planned/unfixed.
- Julia bridge scope understated: julia-engine.Rmd (L393, L1118-1119) and cross-family.Rmd (L290-296) frame the bridge as Gaussian-only / cross-family 'declined', but the R gate admits non-Gaussian phylo (poisson/nbinom2/gamma/beta/binomial) and routes mixed-family pairs in 0.5.0; the nbinom2 bridge error reason is also misdescribed (needs a phylo intercept, not a scale-parity transform).
- capability-and-limits.Rmd tier ledger has two defects pulling opposite ways: it presents univariate-Gaussian q2 phylo/relmat as inference-ready when only the separate BIVARIATE q2 cells are (a potential over-trust of a single-response model), and it denies the 'supported' tier the census assigns to 4 ordinary-RE Gaussian/biv-Gaussian cells (omitting the package's highest-evidence RE cells).
- Only two genuine tier OVERCLAIMS exist and both are low: animal q2 mean-mean gets a report-worthy 95% profile-interval Confidence Eye though it is recovery-only (unlike the coverage-checked phylo/relmat anchors), and lognormal() roxygen levels the mu random slope with the intercept though the slope is recovery-grade. Everything else overclaiming-adjacent is version/example staleness, not capability inflation.
- Developer-vignette / dev-jargon housekeeping: source-map.Rmd omits skew_normal (model_type=17) from all three 'what exists' lists and tweedie from two of them; internal 'Phase 17/18' roadmap jargon leaks into several shipped vignettes (model-workflow.Rmd:600, model-map.Rmd:69, and more).

A user-facing factual reversal in a shipped vignette; the deprecated form still fits with a warning, so it is borderline rather than strictly blocking.
