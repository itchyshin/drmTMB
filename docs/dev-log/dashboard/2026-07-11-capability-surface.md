## Tier snapshot, top gaps, and census-vs-code drift

TIER SNAPSHOT (census _master.tsv, 668 cells / 18 families). STATUS: 283 implemented, 343 rejected_by_design, 42 not_implemented. EVIDENCE TIER: 4 supported, 16 inference_ready_with_caveats, 44 interval_feasible, 66 diagnostic_only, 150 point_fit_recovery, 388 none (58% carry no evidence). The 4 'supported' cells are all ordinary-RE Gaussian/biv-Gaussian, ML, provider=none, with coverage=PLANNED (point-trustworthy, not coverage-verified). The 16 inference-ready cells are all ML (none REML): 8 Gaussian structured anchors + 2 biv-Gaussian phylo/relmat q2 mean-mean + 6 unstructured fixed-effect (beta mu/sigma, nbinom2 mu/sigma, binomial mu, poisson mu). Only 16 cells are genuinely coverage-checked; the 44 interval_feasible cells have a Wald interval that computes but NO Monte Carlo coverage — trust the point, treat the interval as a plausible range.

TOP CAPABILITY GAPS: (1) REML is Gaussian-only — allowed for exactly gaussian + biv_gaussian, blanket-rejected for the other 13 families (R/drmTMB.R:224); even Gaussian REML structured cells cap at diagnostic_only, and no supported/inference-ready cell uses REML. (2) No non-Gaussian family reaches 'supported' — every non-Gaussian family peaks at interval_feasible except beta/binomial/nbinom2/poisson, which reach inference-ready only through unstructured FIXED effects. (3) Structured effects are sparse and lopsided — 8 families have NONE (binomial, lognormal, beta_binomial, skew_normal, tweedie, zero_one_beta, + zi variants degenerate), and most of the rest expose a single provider on a single dpar (gamma relmat-mu, cumulative_logit phylo-mu, student spatial-mu/phylo-nu, beta animal, truncated relmat-hu); full richness exists only for gaussian, nbinom2, poisson (mu±sigma) and biv_gaussian (location). (4) Missing-data support is narrow — miss-response only 6 families (gaussian, biv_gaussian, binomial, poisson, nbinom2, beta); miss-predictor mi() only 5 (gaussian BROAD impute catalogue; poisson/binomial/nbinom2/beta one binary bernoulli predictor each); biv_gaussian and 10 other families reject both. (5) Non-mu random effects are heavily restricted — sigma random rejected in every family except gaussian (full) and nbinom2 (intercept-only); shape/inflation dpars (nu, zi, hu, zoi, coi) are fixed-effect only nearly everywhere; 5 families (binomial, cumulative_logit, skew_normal, tweedie, zero_one_beta) allow NO random effects at all.

CENSUS-vs-CODE DRIFT FOUND: The biggest is biv_gaussian STRUCTURED support. The code-verified gate grader reported 'PHYLOGENETIC LOCATION only; no spatial/animal/relmat.' The actual code implements FOUR structured location-covariance sources on mu — phylo, spatial, animal, AND relmat, one at a time (guard_biv_spatial/phylo/known_mu_terms, R/drmTMB.R:6902-6935) — with q4 all-four blocks extending to sigma1/sigma2 (test-animal-relmat-gaussian.R q4 tests; drm_julia_biv_known_structured_payload admits 'relmat and animal only'; design doc 01-formula-grammar.md:148-149). The census and intervals graders correctly cite phylo + relmat q2; the gate grader undercounted, apparently by reading only the sd_phylo build path (which returns structured=list(phylo_mu=...)) and conflating the ML capability with the REML-scope restriction — under REML the scope genuinely does narrow to phylo location only (R/drmTMB.R:2135-2143), which is the correct nuance. Two lesser notes, not conflicts: (a) the missing-data axis is absent from the census entirely (expected — census dated 2026-07-09 predates the missing-data arc; it lives only in the gate grader); (b) granularity differs — the census lists 18 families while the code dispatches 15 family_types, with zi_poisson/zi_nbinom2/hurdle_nbinom2 routed as poisson/nbinom2/truncated_nbinom2 + an inflation formula (their missing-data eligibility therefore inherits the base family_type gate; no zi/hurdle-specific missing-data guard exists in R/missing-data.R, but the zi-formula × missing-data interaction was not separately coverage-tested by any grader).
